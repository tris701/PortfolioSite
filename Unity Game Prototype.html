Overview

For my senior project in Computer Graphics Technology, I designed a project that would utilize an emerging technology while also allowing me to gain experience working with games. I decided to experiment with using accessible and affordable AI-based motion capture tools to create animations for a game prototype in Unity and Unreal engine. The project served as an opportunity for me to learn about implementing animations into game engines and working with the various tools needed to integrate them. The motion capture tool I used for the project is Plask Motion, an AI-based markerless motion capture method using only a single camera.

Step 1: Planning and Capturing Movements

The prototype was planned to simulate an environment reminiscent of the real world, with fully implemented animations on character controllers for both the player and for NPCs. A variety of different motions were captured on video using three actors of varying body types: a medium-sized male, a medium-sized female, and a tall male. Once the videos were captured, they were uploaded to Plask Motion’s browser tool, where motion data was to be extracted from the videos. Models from Mixamo were downloaded and imported into Maya to be renamed, as their rigs needed to match Plask’s target rig in order for the retargeting process to work. The renamed models were then uploaded to Plask and combined with the motion data from the videos. I dealt with many minor issues with Plask’s browser tool during the process but I was able to contact Plask and resolve them before the project deadline.

Step 2: Animating and Editing Cycles

The finished models and animations were exported as .fbx files and then imported back into Maya. The animation keyframes were adjusted using Maya’s graph editor. The animation length was trimmed and the keyframes were adjusted to loop or invert depending on the type of motion. The idle and walk animations were the most important to get right, as they needed to loop smoothly and frequently. Translation keys were also removed from each rig’s hips to stay aligned with Unity’s character controller, which would dictate the character’s movement.

Step 3: Implementing Animation in a Character Controller

The next step was to import the finished animations into Unity, where a character animation controller was made to allow the character to cycle between the various movement animations. A large amount of time was spent trying to implement the animations into Unreal Engine 5 at first, but after encountering many issues with Unreal Engine’s character controllers, I decided to use Unity. The process of importing and implementing .fbx files into a character controller was much easier in Unity from my experience, Unity’s Loop animation and loop pose helped make transitions appear smoother. Animations other than the walk, jump, and idle were implemented into their own animation controllers and applied to static NPCs, which played animations on loop.

Despite Unity having the easier import process, this stage caused me the most problems during the project, as there were many issues trying to get the animations to work. They may have been due to compatibility issues when importing/exporting, as the original process I used to import/export them did not include the rig renaming step. Once I figured out the correct process for getting the animations from Plask and importing them into Unity, I recreated all of the animations with a compatible rig and was able to apply them to animation controllers.

A scene was created to showcase various animations from our motion capture recordings. The game environment was created using assets from the Unity Asset store under an Extension Asset license. The final demo included a playable character with a complete set of movement animations and static NPCs. The demo was presented to CGT faculty as a part of my Senior Project.

Conclusion

This project gave me the opportunity to work with an emerging technology in animation and create a game prototype with it. As an animator looking to work in the game industry, this project was my first experience working with original animations in Unity, and I learned a lot about the process behind animating game characters. The project also taught me a lot about project planning and management, allowing me to create a project pipeline and put it into action.
